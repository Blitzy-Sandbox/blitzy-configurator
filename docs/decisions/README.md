# StrikeForge Backlog — Decision Log

## Purpose

This document is the single source of truth for the non-trivial authoring decisions that shaped the StrikeForge backlog. The epic and story files under `/tickets/epics/` and `/tickets/stories/` deliberately carry no rationale in their bodies — they state goals, personas, capabilities, and acceptance criteria only. Every answer to the question "why did we choose this scope, this split, or this estimate?" lives in the table below, and the ticket files cross-reference back to this log whenever a decision materially shaped their content.

## How to Read This Log

The decision log is a Markdown table with four columns: **Decision** (what was chosen, with ticket ID references whenever the choice shaped a specific epic or story), **Alternatives** (the options that were considered and rejected), **Rationale** (the reasoning that made the chosen option preferable), and **Risks** (the residual exposure carried by the choice, with mitigation where applicable). Other artifacts that need to cite a specific decision do so by quoting a distinctive leading phrase from the **Decision** cell — for example "the decision titled 'No bidirectional traceability matrix is emitted for this deliverable'" — so that citations remain stable when rows are added or reordered. A new row is added whenever a competent reviewer could reasonably have chosen differently; every such choice must be traceable here rather than buried in commit messages or ticket bodies.
